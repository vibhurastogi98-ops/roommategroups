# RoommateGroups

RoommateGroups is a Cloudflare Workers app for finding rooms, rentals, roommate matches, and local housing communities.

The production stack is:

- Cloudflare Workers API in `backend/worker.ts`, built with Hono.
- Cloudflare D1 database, exposed to the Worker through the `DB` binding.
- Cloudflare R2 object storage, exposed to the Worker through the `BUCKET` binding.
- Vanilla-JS single-page app in `web/src`, built with Vite.
- Capacitor 8 native app in `mobile/`, using the built web app from `web/dist`.

## Commands

Install dependencies:

```bash
npm install
```

Run the web app:

```bash
npm run dev:web
```

Run the Worker locally:

```bash
cd backend
npx wrangler dev
```

`web/vite.config.js` proxies `/api` and `/r2` to the local Worker on `127.0.0.1:8787`.

Build the web app:

```bash
npm run build:web
```

Deploy the Worker and built web assets:

```bash
npm run deploy
```

Sync and open native projects:

```bash
npm run build:android
npm run build:ios
```

Run D1 migrations from the repo root:

```bash
npx wrangler d1 execute roommatedb --local --file=./database/schema.sql --config ./backend/wrangler.toml
```

## Notes

- `backend/worker.ts` is the application API entry point and exports the Hono Worker.
- `backend/wrangler.toml` binds D1 as `DB`, R2 as `BUCKET`, and serves assets from `../web/dist`.
- `web/dist` is generated by `npm run build:web`.
- `mobile/capacitor.config.json` points Capacitor at `../web/dist`.
- Root `wrangler.toml` is a deploy shim for Cloudflare dashboard builds that run `npx wrangler deploy` from the repository root.
- Secrets stay in `.env` and `backend/.dev.vars`; both are ignored.
- `dev/` contains scratch scripts for manual diagnostics.
